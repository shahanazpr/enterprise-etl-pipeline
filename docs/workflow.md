# Project Workflow

## Development Workflow

1. Extract data from the API.
2. Validate the data using Pydantic.
3. Transform the data.
4. Load the data into the database.
5. Log the execution.
6. Run unit tests.
7. Push changes to GitHub.

---

## Git Workflow

- Clone the repository.
- Create a feature branch.
- Develop the feature.
- Commit changes.
- Push the branch.
- Create a Pull Request.
- Merge after review.

---

## Daily Development Process

Developer → Feature Branch → Pull Request → Review → Merge into Main
